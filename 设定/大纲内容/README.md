# 大纲内容导航

> 本目录负责“故事怎样推进、真相何时揭露、伏笔何时回收”。它可以安排设定的出场顺序，但不能重新定义世界级事实。

## 一、跨卷文件

| 文件 | 状态 | 职责 |
|---|---|---|
| [总纲.md](总纲.md) | 剧情总纲/概览 | 故事承诺、七卷功能、人物成长与结局方向；不是项目设定宪章 |
| [故事线推进表.md](故事线推进表.md) | 跨卷执行源 | 七条人物与剧情主线按卷推进 |
| [世界秘史推进表.md](世界秘史推进表.md) | 揭露节奏源 | 世界真相何时被谁知道；真相本身回到世界观等专项定稿源 |
| [跨卷伏笔追踪表.md](跨卷伏笔追踪表.md) | 追踪视图 | 登记伏笔埋设、升级与回收状态 |
| [反面vs绝区零空洞-设定对照参考.md](反面vs绝区零空洞-设定对照参考.md) | 非定稿参考 | 只用于概念对照与差异化检查，不进入裁决链 |
| [第一卷/](第一卷/README.md) | 当前卷执行目录 | 第一卷进度、事件链、口径和运行态缓存 |

## 二、推荐读取路线

- 查全书故事承诺：剧情总纲 → 故事线推进表。
- 查世界真相如何揭露：世界观设定 → 世界秘史推进表。
- 查某个伏笔：跨卷伏笔追踪表 → 对应卷事件链 → 当前写作状态。
- 写当前卷：直接进入对应卷 README，不从跨卷概览反推现行章号。

## 三、边界

1. `设定/设定文件.md`是项目宪章；本目录的`总纲.md`是剧情总纲，两者不再共用同一职责。
2. 世界观、能力和术语的完整定义必须回到对应专项定稿源。
3. 第一卷规划文件中的 `ChN` / `第N章` 默认是节拍编号；实际正文章号只认第一卷当前写作状态。
4. 对照参考、缓存和历史节拍不得覆盖当前硬口径、当前写作状态或分卷大纲。
