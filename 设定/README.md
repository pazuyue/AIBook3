# 《三月女神今天也只是普通学生》设定导航

> 本文件是设定区的唯一导航入口，只负责“去哪里找”和“发生冲突时怎么裁决”，不新增剧情事实，也不覆盖任何专项定稿源。
>
> 为保持现有工作流稳定，本轮整理不移动、不改名高引用文件；目录分类均为逻辑分类。

## 一、先从哪里开始

| 任务 | 推荐入口 | 继续读取 |
|---|---|---|
| 写下一章 | [第一卷导航](大纲内容/第一卷/README.md) | 当前硬口径 → 当前写作状态 → 当前章写作卡 → 分卷大纲/事件链 → 必要专项设定 |
| 调整全局设定 | [项目宪章](设定文件.md) + [单一来源索引](写作规则单一来源索引.md) | 找到对应字段定稿源后修改，不在概览或大纲里另立版本 |
| 处理当前冲突或禁写 | [当前硬口径](当前硬口径.md) | 若与项目宪章冲突，停止并人工裁决 |
| 设计战斗或能力 | [神秘设定](神秘设定.md) + [战斗设定](战斗设定.md) | 角色能力文件 → [核心技与解放归档](核心技与解放归档.md) / [术式吟唱归档](术式吟唱归档.md) |
| 查人物 | [人物设定导航](人物设定/README.md) | 人物总表 → 能力总表 → 角色专项文件 |
| 查七卷结构与伏笔 | [大纲导航](大纲内容/README.md) | 剧情总纲 → 故事线/秘史线 → 跨卷伏笔 → 当前卷 |
| 查固定叫法 | [术语口径统一表](术语口径统一表.md) | 术语表只裁决词义与写法，完整机制回到对应专项定稿源 |

## 二、裁决层级

导航文件没有裁决权。出现冲突时按以下顺序处理：

1. **项目宪章层**：[设定文件.md](设定文件.md)  
   只裁决题材定位、主角身份、核心创作承诺与全局方向。
2. **当前增量裁决层**：[当前硬口径.md](当前硬口径.md)  
   裁决当前卷仍有效的覆盖项、禁写项和临时边界；不得反向改写项目宪章。
3. **字段归属层**：[写作规则单一来源索引.md](写作规则单一来源索引.md)  
   只回答“哪个文件定稿”，不直接定义事实。
4. **专项定稿源层**：世界观、神秘、战斗、人物、组织、术语、归档等专项文件。  
   同一字段只在指定文件维护完整版，其他文件只保留摘要和引用。
5. **卷级执行层**：当前写作状态、分卷大纲、事件链、口径卡片、写作设定包。  
   可以收紧本卷可见范围，但不能重新定义全局机制。
6. **派生与参考层**：节奏表、统计表、缓存、对照参考。  
   只能辅助执行，不能参与设定冲突裁决。

若低层文件与高层文件冲突，以高层为准；若专项文件之间冲突，先查单一来源索引。项目宪章与当前硬口径冲突时，不自动合并。

## 四、维护规则

1. **先找归属，再改内容**：新增或修改事实前，先查单一来源索引。
2. **完整版只留一份**：其他文件写一句摘要和项目根相对路径，不复制整张规则表。
3. **现行章号只认状态文件**：第一卷规划文件中的 `ChN` / `第N章` 默认解释为规划节拍；实际正文章号只以“第一卷当前写作状态”的章↔节拍映射为准。
4. **归档记录已定稿连续性**：能力设计先写角色文件，正式使用或确定不可改后再进入对应归档。
5. **参考资料必须显式降级**：对照作品、旧节拍和缓存不得反向覆盖现行设定。
6. **路径保持稳定**：机器读取统一使用项目根相对路径（例如 `设定/神秘设定.md`）；不以行号作为长期定位。

## 五、文件状态标签

| 标签 | 含义 |
|---|---|
| 项目宪章 | 全局创作承诺，改动需人工确认 |
| 专项定稿源 | 某类字段的唯一完整版 |
| 卷级执行源 | 当前卷实际写作依据 |
| 连续性归档 | 已定稿名称、原文与复用状态 |
| 概览/派生视图 | 方便阅读的摘要，不独立裁决 |
| 历史/参考 | 可借鉴，不得覆盖现行口径 |
| 运行态缓存 | 可被流程覆盖，不是设定源 |
