# /split-chapter

Description: 按 Trae 新章节规划 skill 和定稿入库 skill 执行人工裁决的拆章/合章操作。

## Source of truth
- 读取并服从 `.trae\rules\三月女神新章节规划skill.md` 的「拆章与合章执行口径」章节。
- C 层入库清理按 `.trae\rules\三月女神定稿入库skill.md` 的「拆章入库清理口径」执行。
- 全文件链同步以 `设定\当前硬口径.md`、当前卷状态、事件链、节奏表为准。

## Steps
1. 确认用户已明确裁决拆分方案：原章节号 → 新章节号映射、每章事件边界和章尾钩子。
2. 确认下游影响范围：原章节号之后的所有章节号偏移量。
3. 确认当前无正在执行的 `/write`、`/finalize` 或检查命令。
4. 若用户未给出方案，可协助分析建议，但不得自行决定并执行拆分。
5. 按 Trae skill 的 A 层（计划层）清单逐项执行，每完成一项标注 ✅。
6. 若涉及正文拆章，继续执行 B 层（正文层）操作。
7. 若涉及定稿后拆章，调用 `/finalize` 执行 C 层（入库层）清理。
8. 拆章完成后，按当前状态提示恢复路径。
