# /split-chapter

Description: 按 Trae 新章节规划 skill 执行人工裁决的节拍分流、拆章/合章操作；涉及定稿后结构变更时，再按定稿入库 skill 执行 C 层清理。

## Source of truth
- 读取并服从 `.trae\rules\三月女神新章节规划skill.md` 的「拆章与合章执行口径」章节。
- C 层入库清理按 `.trae\rules\三月女神定稿入库skill.md` 的「拆章入库清理口径」内部调用执行。
- 结构影响范围以 `设定\当前硬口径.md`、当前卷状态、事件链、节奏表和章↔节拍映射表为准。

## Steps
1. 先判断是否只是写作时的节拍承载调整：尚未生成需要切割的正文文件时，不执行 `/split-chapter` 全流程，只由 `/chapter-plan` 声明对应节拍，并在 `/finalize` 时登记章↔节拍映射。
2. 若需要改动已有正文文件或已定稿章节，确认用户已明确裁决拆分方案：原章节/正文文件 → 新章节/正文文件映射、每章事件边界和章尾钩子。
3. 确认影响范围：章↔节拍映射、伏笔实际章号、正文文件名、缓存条目、下游正文章号偏移（如有）。
4. 确认当前无正在执行的 `/write`、`/finalize` 或检查命令。
5. 若用户未给出方案，可协助分析建议，但不得自行决定并执行拆分。
6. 按 Trae skill 的 A 层（计划层）清单逐项执行，每完成一项标注 ✅。
7. 若涉及正文拆章，继续执行 B 层（正文层）操作。
8. 若涉及定稿后拆章，调用定稿入库 skill 的 C 层口径执行旧向量清理、重新入库和 references 同步。
9. 拆章完成后，按当前状态提示恢复路径。
