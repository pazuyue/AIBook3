# 三月女神专属续写技能（Trae 正式版 v2）

## 概述
专为《三月女神今天也只是普通学生》定制，按当前有效大纲续写、人设绝对统一、不OOC、不崩战力、不提前曝光身份、神秘学基调、日常+危机双线。

## Master 角色定位

本 skill 在 Master 总导演调度制中承担：正文写手 + 设定检索员。

职责：
1. 按 `/write-brief` 和当前权威源生成正文，只输出可进入正文的小说段落。
2. 执行写前本地文件和 Qdrant 记忆检索，发现冲突时以高优先级源裁决。
3. 执行 `/word-check` 的字数和完成度闸门，判断是否能进入写后修订。

不负责：
1. 不生成章节方案，不替代 `/chapter-plan` 和 `/write-brief`。
2. 不做定稿入库，不处理 `/finalize` 或 `/upload-*`。
3. 不在正文生成中自行新增关键设定、伏笔结论或世界观裁决。

被 Master 调用时，只处理正文生成和必要检索维度，不自动扩展到审稿、重写或定稿职责。

## 执行摘要

| 触发 | 先做 | 必查 | 输出 |
|------|------|------|------|
| `/write` | 读 L1 状态、写作包、口径卡片、当前卷 `write_brief_cache.md`、最近正文、`chapter_plan_cache.md` | `/write-brief` 优先于计划缓存；战斗/术式查 `power`，世界观/古脉/秘史查 `world`；Qdrant 只召回，本地设定裁决 | 只输出小说正文 |
| `/word-check` | 统计当前章去空白字数 | 分场计划、章尾钩子、章节号 | 低于 4000 字只能继续补写 |
| `/finalize` / `/upload-*` | 停止本流程 | 转交 `sanyue_finalize` | 不在本 skill 入库 |
| Qdrant 不可用 | 启用本地文件降级 | 人设、章节母版、最近正文不可跳 | 标注“本轮未使用 Qdrant 召回” |

快速裁决顺序：AIBook3 `设定/当前硬口径.md` > AIBook3 当前卷状态/写作设定包/口径卡片/正文定稿 > 人工 `/write-brief` > 当前章节计划缓存 > Qdrant 召回 > references 降级参考。主角伪装态、术式吟唱、异常反馈只引用定稿源和《三月女神写作风格skill.md》，不在本文件重复定义。

## 前置执行原则
执行任何续写前，优先按本文件的分层检索流程和《三月女神写作风格skill.md》的 L1/L2/L3 分层写作流程工作。当前进度属于第一卷时，必须先读取《设定/大纲内容/第一卷/第一卷写作设定包.md》，再按缺口补查全局设定和 Qdrant。

第一卷续写还必须读取《设定/大纲内容/第一卷/第一卷当前写作状态.md》，用于确认当前章节、上一章结尾、主角状态、活跃伏笔和输出前自检。

第 N 卷通用 L1 规则：若 writing_state、当前卷状态文件或章节号显示已进入第二卷及以后，按当前卷目录读取同结构文件：`第N卷当前写作状态.md`、`第N卷写作设定包.md`、`第N卷口径卡片.md`、`第N卷事件链.md`、`第N卷章节目录版节奏表.md`、当前章节细纲、当前卷 `write_brief_cache.md` 和当前卷 `chapter_plan_cache.md`。若对应文件尚未建立，标注为“口径缺口”，回退读取《故事线推进表》《总纲》和全局设定，不得套用第一卷专属禁写和章节母版。

卷号推断规则：优先使用 writing_state/current 状态文件中的当前卷；其次用章节所在路径；再次用章节号与已定稿目录推断。若当前章节号超出第一卷范围，不得继续硬套 `第一卷/` 路径，必须改按当前卷目录或标注缺口。

若当前环境存在 `novel-craft-sanyue` 技能，可以把它作为修文阶段的去AI化参考；若不存在，不得因为缺少该技能而停止续写，直接使用本文件和《三月女神写作风格skill.md》完成正文。

## 续写前置流程（分层检索，自动执行）

### L1 必载层（直接读取文件，不经过检索）
0. **读取当前硬口径**：优先读取 `设定/当前硬口径.md`。这是当前执行口径短卡，用于裁决旧规则、旧缓存、Qdrant/references 召回和大纲残留。若其与 `chapter_plan_cache.md`、`rewrite_plan_cache.md`、Qdrant 召回或 references 冲突，以 `设定/当前硬口径.md` 为准；若与全局不可变硬设定冲突，停止并提示人工裁决。
1. **优先读取 AIBook3 当前卷状态文件**：第一卷读取 `设定/大纲内容/第一卷/第一卷当前写作状态.md`；第二卷及以后读取同结构 `第N卷当前写作状态.md`。这是当前写作进度、上一章结尾、主角状态、活跃伏笔和本章目标的最高优先级来源。
2. **读取当前卷写作设定包/口径卡片**：第一卷读取 `设定/大纲内容/第一卷/第一卷写作设定包.md`；第二卷及以后读取同结构写作设定包和口径卡片。
3. **读取当前卷 `write_brief_cache.md`**：优先使用用户本轮提供的 brief；若未提供，读取当前卷 `write_brief_cache.md` 的当前活跃 brief。第一卷固定路径为 `设定/大纲内容/第一卷/write_brief_cache.md`。缺少当前章节 brief 时停止并提示，不进入正文生成。`/write-brief` 用于建立场景任务卡，优先于计划缓存。
4. **读取 `NovelFlow/references/writing_state.md` 只作同步校验/降级参考**：若与 AIBook3 当前卷状态冲突，以 AIBook3 当前卷状态为准，并提示 references 可能未同步。
5. **读取 `NovelFlow/references/foreshadow_tracker.md`**：伏笔追踪表（已埋未收的高优先级伏笔）
6. **读取当前卷 `chapter_plan_cache.md`**：若存在当前章节号的最近 `/chapter-plan` 或 `/quality-card`，只作为计划参考；不得覆盖 `/write-brief` 的四个创作指标、一页结构、不可改事实和最多 3 条本章禁区；若与权威源冲突，先提示冲突，不继续按错误计划续写
7. 若 AIBook3 当前卷状态文件不存在，仍需按卷号推断规则读取当前卷写作设定包与口径卡片；无法判断卷数时再进入L2

### L2 核心检索层（Qdrant语义检索，精准召回）
用当前上文做语义检索，必须调用 `search_doc` 并优先使用 NovelFlow Qdrant 新增过滤字段：`content_type`、`source_path`、`source_file`、`volume`、`visibility_scope`、`authority_gte`、`is_final_chapter`、`is_summary`。默认每次 `limit=3`，最近正文可 `limit=5`。

> 以下检索模板字段已按 NovelFlow Qdrant MCP server v2.0.0 inputSchema 验证（2026-05-22），`content_type`、`source_path`、`volume`、`authority_gte`、`is_final_chapter` 等过滤字段均受支持，`collection_key` 与 `collection` 均可传参。

Qdrant 只负责语义召回和补查，不作为最终裁决源。检索结果与 AIBook3 本地 `设定/`、当前卷状态、正文定稿冲突时，以本地权威源和更高 `authority_level` 为准。

若 Qdrant、MCP 或向量服务不可用，不得假装已经召回；改用本地文件降级，并在回复中标注“本轮未使用 Qdrant 召回”。降级规则：
- L2-1 人设硬约束不可跳：必须直接读本地人物设定、主要角色能力发展表和出场角色能力发展设定替代。
- L2-2 章节母版不可跳：必须直接读当前卷事件链、章节目录版节奏表、当前章节细纲、`write_brief_cache.md` 和 `chapter_plan_cache.md`；其中 `write_brief_cache.md` 优先于计划缓存。
- L2-3 最近正文不可跳：必须读取 AIBook3 工作区上一章/上两章正文原文替代，不得用摘要替代正文事实。
- L2-4 伏笔可降级：优先读 `foreshadow_tracker.md` 和当前写作状态；若缺失，标注伏笔信息不足，不新增重大回收。
- world/power 检索可降级为直接读《世界观设定》《神秘设定》《战斗设定》和相关人物能力设定。

#### L2-0 第一卷裁剪层确认（第一卷进度必查）
若当前进度属于第一卷，先查第一卷写作包，作为 Qdrant 召回中的最高优先裁剪层：
```json
{
  "collection_key": "chapter",
  "query": "第一卷 可见设定 正向写作口径 当前硬口径 角色阶段 战斗口径 身份隐藏 回查规则",
  "content_type": "volume_pack",
  "volume": 1,
  "limit": 3
}
```
若当前进度不属于第一卷，跳过本裁剪层，直接进入 L2-1。

#### L2-1 人设硬约束检索
根据当前场景出场人物、上一章结尾人物状态和本章大纲，检索 `char` 集合：
```json
{
  "collection_key": "char",
  "query": "出场角色 人设 外貌 身份边界 能力阶段 关系变化 普通学生态 女神态区分 主角伪装态 主动降档 体内压制禁写",
  "content_type": "character_setting",
  "limit": 3
}
```

若涉及主角伪装、性别适应、身份风险，query 必须包含“哥伦比娅 普通学生态 黑为底 根黑尾紫 女性肉身适应 隐藏身份”。

#### L2-2 当前章节母版检索
`chapter` 集合检索必须拆成“当前章母版”和“最近正文”两类。当前章母版检索使用 `writing_state.md` / `第一卷当前写作状态.md` 中的当前章节标题、本章大纲要点、核心事件作为 query，优先查事件链：
```json
{
  "collection_key": "chapter",
  "query": "当前章节标题 本章大纲要点 核心事件 暗线推进 月相 反面状态 日常线 悬念落点",
  "content_type": "event_chain",
  "volume": 1,
  "limit": 3
}
```

再查章节细纲/节奏表：
```json
{
  "collection_key": "chapter",
  "query": "当前章节标题 月相 反面状态 核心事件 暗线推进 日常线 悬念落点 节奏表 章节母版",
  "content_type": "chapter_outline",
  "volume": 1,
  "limit": 3
}
```

若当前章节母版信息仍不足，再以 `source_file` 精准补查：
```json
{
  "collection_key": "chapter",
  "query": "当前章节 母版 事件链 分卷大纲 节奏表",
  "source_file": "{当前卷}事件链.md",
  "volume": "{当前卷号}",
  "limit": 3
}
```
`{当前卷}` 与 `{当前卷号}` 必须替换为实际卷名和数字；占位符未替换则该补查步骤视为失败，必须先按卷号推断规则确认当前卷。

#### L2-3 最近正文定稿检索
若已有上一章或前两章定稿，必须按章节号传入 `chapter_gte` / `chapter_lte`，并加 `is_final_chapter=true`，只召回最近两章正文定稿；不要只靠语义检索从大纲里猜前文：
```json
{
  "collection_key": "chapter",
  "query": "上一章结尾 当前时间点 人物站位 情绪余波 对话关系 未解决动作",
  "chapter_gte": "{上上一章号}",
  "chapter_lte": "{上一章号}",
  "is_final_chapter": true,
  "limit": 5
}
```
`{上上一章号}` / `{上一章号}` 必须替换为实际数字；占位符未替换则该检索步骤视为失败，必须先回查 writing_state、章节目录或正文路径确认章节号，不能带占位符调用检索。

若正文定稿尚未通过 `/finalize` 单章入库，则直接读取 AIBook3 工作区当前卷正文目录（第一卷为 `正文/第一卷`）对应章节原文，不得用大纲或摘要替代正文事实。

#### L2-4 伏笔与谜题检索
检索 `foreshadow` 集合时优先召回追踪表和高权威伏笔来源：
```json
{
  "collection_key": "foreshadow",
  "query": "当前章节 活跃伏笔 未解决谜题 规则载体 暗线推进 回收风险",
  "authority_gte": 80,
  "limit": 3
}
```

第一卷进度下，L2 检索应优先围绕《第一卷写作设定包》和当前章节母版补足缺口，不把后期全局设定作为常规输入。

检索结果包含 `score`、`file`、`source_path`、`source_file`、`content_type`、`authority_level`、`visibility_scope`、`section_title`、`chapter_number`、`chapter_title`、`is_final_chapter`、`is_summary` 与 `content` 时，写作前必须先判断来源：正文定稿（`is_final_chapter=true` / `content_type=final_chapter`）> L1状态文件（`writing_state` / `foreshadow_tracker`）> 第一卷写作包（`volume_pack`）> 事件链（`event_chain`）> 章节细纲/节奏表（`chapter_outline`）> 人物/世界/战斗定稿设定 > 章节摘要（`chapter_summary`）> 通用大纲/笔记。同一事实冲突时，以 L1、正文定稿和更高 `authority_level` 为准。

时间线硬约束：凡涉及“第几天 / 第几夜 / 昨天 / 今天 / 前夜 / 次日”等相对时间，必须优先读取并对齐最近两章正文定稿（`chapter_body.md` 或工作区当前卷正文目录对应章节）。大纲、事件链、节奏表中的相对时间只视为草案标签；若与正文定稿冲突，必须判定为旧大纲残留，不得沿用。续写前必须把上一章结尾状态和当前章开头时间点写入场景任务卡。

### L3 补充检索层（按需；战斗/术式场景强制）
若L2召回内容不足以覆盖当前场景，补充检索：
1. 第一卷进度优先回到《第一卷写作设定包》《第一卷分卷大纲》确认本卷战斗与敌方单位口径
2. `world`集合：仅补查世界观/神秘学体系/百城八律/普通社会生活成本/反面边界/秘史揭露梯度/场景细节缺口，优先 `content_type=world_setting`
3. `power`集合：仅补查文风/能力体系/战斗设定缺口，优先 `content_type=power_setting`，文风才查 `content_type=writing_style`

若当前场景出现战斗、演武、术式发动、能力对抗、封堵、护场、拆解、追击、危机处理，先检查《第一卷写作设定包》是否已覆盖敌方单位、队友阶段和收口口径；不足时再检索 `power` 集合，至少覆盖：
- 战斗规模与战力层级
- 神秘核心、术式、展开方式的三层关系
- 普通术式吟唱层级：全吟唱 / 节省吟唱 / 无吟唱
- 展开用语边界：只用于领域、半展开、女神态、视界或高位战场结构

第一卷灰白异常、异核/空洞核心/反面核心、术式名/吟唱/展开等术语边界，统一查《设定/术语口径统一表.md》。普通术式吟唱层级、无吟唱例外和战斗落地写法，统一查《设定/战斗设定.md》和《设定/大纲内容/第一卷/第一卷口径卡片.md》。

### 检索结果约束
- L1内容作为**最高优先级约束**，L2次之，L3最低
- 第一卷进度下，《第一卷写作设定包》是 L1 裁剪层，负责限制当前卷可见设定与正向写作口径；当前卷禁写只读 `设定/当前硬口径.md`，全局设定只做底层裁决和缺口补查
- 章节事件母版以当前卷事件链、章节目录版节奏表和对应章节细纲为准；`writing_state.md` 只记录当前进度和近期状态。若本章要点/近期预览与事件链冲突，必须判定为状态文件滞后，不能覆盖事件链
- 续写前必须用当前章节号核对章节母版：标题、月相、反面状态、核心事件、暗线推进、日常线、悬念落点。场景任务卡必须服从该章节母版
- 姓名、古脉身份、外貌发色、能力底层、身份边界、核心关系等**不可变人设**，以 L1硬约束 与 `char`集合人物设定为准；若最近正文、章节摘要或大纲片段与人物设定冲突，必须判定为旧正文残留或特殊形态误用，不得沿用。
- 普通学生伪装态与女神态外观必须分开判断：银白/冰蓝等发色只可用于明确展开霜月女神态的场景；当前学院日常普通学生态必须服从人物设定中的"黑为底、根黑尾紫"。
- 若当前卷已存在明确章节母版，优先服从该卷的章节目录、节奏表与已确认事件链
- 当前第一卷以《第一卷当前写作状态》列出的“第一卷口径封版源（唯一执行源）”为最高执行清单；章节母版再按事件链、分卷大纲、节奏表和细纲互证。

## 续写生成补丁

生成正文时必须调用《三月女神写作风格skill.md》的"场景任务卡""二轮生成法""正文工艺规则"三项规则。

简化执行顺序：
1. 先用检索结果确认当前章节目标、出场人物、活跃伏笔和上一章结尾状态
2. 再调用《三月女神写作风格skill.md》的“场景任务卡”六项和战斗扩展项
3. 首稿只管场景成立、人物行动、冲突推进、画面清楚
4. 修文再处理 AI 腔、句式重复、感官不足、设定说明、爽点不够、钩子偏弱
5. 水段判定按《三月女神写作风格skill.md》场景任务卡六项校验

长度不再用固定字数硬卡。默认按完整场景单元控制篇幅：小续写 800-1500 字，常规场景 1800-3000 字，单次正文输出 2000-3000 字最稳，章节主体 4500-6000 字；大战和卷末章按事件密度扩展。修文密度检查按《三月女神写作风格skill.md》修文轮执行。禁止在没有场景任务卡的情况下单次生成整章。

`/write` 输出后必须先过 `/word-check`。`/word-check` 只做字数和完成度闸门：统计当前章去空白字数，核对是否完成分场计划、章尾钩子和当前章节号。低于 4000 字判定为未完成章节，只能继续 `/write` 补完当前章；4000-4500 字判定为偏短，优先补厚；4500-6000 字才视为正常章节主体。低于 4000 字时禁止进入 `/self-revise`、`/review`、`/finalize` 或下一章。

战斗、演武、考核、封堵、救援等场景必须在场景任务卡里写明：场景类型、参战人数、敌方单位、主高光位、托举位、收束位、胜利条件、失败风险。正文一旦写出比赛规则或任务规则，后文不得临时改赛制、改人数、改胜负条件。若章节母版写“哥伦比娅一打五，白砚河唯一能接三招”，正文禁止改成“五人一组自由组队”；确需改赛制时，必须先同步修改章节母版与当前写作状态。
