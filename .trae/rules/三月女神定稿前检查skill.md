# 三月女神定稿前轻量检查（Trae 规则版）

## 定位

专为定稿前快速复查设计，每次只针对单一维度。不替代 `/review` 完整审稿，不写正文，不入库。

## Master 角色定位

本 skill 在 Master 总导演调度制中按命令承担不同角色：

1. `/scene-check`：战斗演出师 + 神性描写师 + 战力审计员。
2. `/continuity-check`：人设监督 + 伏笔控制器。
3. `/setting-term-check`：世界观总编 + 设定检索员 + 战力审计员。
4. `/hardword-check`：AI腔清理师 + 文风润色师。

职责：
1. 定稿前按单一维度快速检查问题，输出原位建议和下一步分流。
2. 在高风险章节中承接 Master 指派的专项检查，不扩展为完整审稿。

不负责：
1. 不直接修改正文，不做定稿入库。
2. 不替代 `/review` 的综合审稿，也不替代 `/self-revise` 的整章修文。

被 Master 调用时，只处理本次命令对应角色维度，不自动串联其他检查。

**执行前必查**：`设定/当前硬口径.md`；若正文与当前硬口径冲突，优先判定为旧口径回潮，不继续检查。

## /scene-check 场景通读

只按场景快速检查是否能定稿。

### 职责边界

- 只判断场景是否能定稿，不直接修改正文，不做完整审稿，不替代 `/review`。
- 用户给单场，只检查该单场，不评价整章章尾；用户给整章，逐场检查，并额外检查场景之间的承接。
- 日常场只判断是否有目标、关系变化和场景结果；对话人味、生活语用和日常质感交给 `/daily-style-check`。
- 发现局部句段问题，下一步转 `/fix-prose`；发现连续性冲突，下一步转 `/continuity-check`；发现剧情目标、事件结果或章尾必要性问题，下一步转 `/review` 或重新做章节方案。

### 目标解析

1. 用户给出文件路径：读取并检查该文件。
2. 用户贴出正文：只检查贴出的正文。
3. 用户没有给目标：默认读取当前写作状态里的当前章；若无法定位章节，先询问，不猜测文件。

重点只查七件事：
1. 每场有没有明确地点、人物、目标和结果
2. 日常场有没有关系推进或暗线轻触
3. 设定显影场有没有具体痕迹、角色判断和可见后果
4. 战斗场有没有站位、动作线、受击结果和收口
5. 主角出手后有没有合理遮掩口径
6. 章尾是不是让下一章变得必要
7. 场景之间的时间、地点、人员转移是否有可见承接

输出格式：

```text
【场景通读结论】

【逐场检查】
1. 场景名/范围：
   - 地点、人物、目标、结果：
   - 关系变化/暗线推进：
   - 神秘痕迹/战斗动线：
   - 问题：
   - 原位建议：

【必须修】

【问题分流】
- 可直接 /fix-prose：
- 需 /continuity-check：
- 需 /review 或重做章节方案：

【可保留】

【下一步】
```

## /continuity-check 连续性核对

只核对当前章与前文已发生事实是否冲突。默认读取当前章、上一章、当前写作状态和已定稿章节摘要；正文与状态文件冲突时以正文定稿为准。

重点检查：
1. 人物关系阶段：是否把已经认识、合作、信任过的人写回陌生状态
2. 居住/队伍状态：宿舍人数、床位、分组、同行人员是否前后一致
3. 时间与月相：当前章月相、上一章结尾和本章开头是否顺
4. 关键行为次数："第一次""第二次""从未""终于"等绝对表述是否被前文打脸
5. 能力与信息暴露：角色知道/不知道什么，是否越过当前认知边界
6. 物品和伤痕状态：谁拿着什么、伤痕是否消失、记录是否已归档
7. 空间与场景连续：上一章结尾地点是否与本章开头一致；跨场景转场是否有合理衔接

输出格式：

```text
【连续性结论】

【必须改】
1. 位置：
   - 冲突事实：
   - 前文依据：
   - 原位改法：

【建议微调】
1. 位置：
   - 风险：
   - 原位改法：

【可保留】

【下一步】
```

## /hardword-check 硬词套路句

定稿前硬词、套路句和 AI 腔复查。默认检查当前章；指定章节则只检查指定章节；用户贴正文则只检查贴出的正文。

**权威源**：硬词表以《设定/文风设定.md》"定稿硬词与套路句检查附录"为准；术语类命中再按《设定/术语口径统一表.md》判断。

处理原则：
1. 术语类硬词按《设定/术语口径统一表.md》判断；该表未覆盖的新冲突，标注为"口径缺口"
2. "不可名状/理智崩坏/疯狂低语"按克系化风险处理，原则上必须改
3. "全场安静/顿了一下/沉默了一下"不是绝对禁词，但单章重复时必须压缩或换成具体声音、动作、仪器反应
4. "某种/仿佛/像是/意识到/复杂/无法形容"不是禁词，只在密集出现、句子偷懒或抽象空泛时要求改
5. "更大的危机正在逼近"一类总结式悬念改成具体痕迹、声音、记录、灯光或人物反应
6. "拆"不要拿来描述抽象情绪、快乐、气氛或人物关系；战斗、术式、物品和技术语境里可按上下文保留
7. "发动神秘""全展开""节省展开""无展开"按术语回潮处理
8. "体内压制力量""身体深处涌动""按回去""钻出来""反噬失控"用于主角伪装态时，按无敌流口径跑偏处理；改为主动降档、输出收束、能力外观遮掩
9. "过分端正""用力太均匀""节奏太稳""没有半点迟疑""找不出半点摸索感""不该有的分辨率""像精密仪器""像程序""像机器""动作校准""步幅""重心"用于日常、宿舍、变身适应或生活动作时，按诊断式旁白处理；改为具体动作结果、旁人反应和角色补救
10. 出现"魔力、魔力源、魔力模型、月相供能、月蚀期魔力"等词，转入 `/setting-term-check` 复核

输出格式：

```text
【硬词检查结论】

【必须改】
1. 位置：
   - 命中词：
   - 问题：
   - 原位改法：

【建议微调】
1. 位置：
   - 命中词：
   - 问题：
   - 原位改法：

【可保留】

【下一步】
```

## /setting-term-check 设定术语

只核对正文、细纲或设定说明里的体系术语是否与定稿源一致。

**权威源**：《设定/术语口径统一表.md》《设定/神秘设定.md》《设定/战斗设定.md》《设定/以太体系设定.md》《设定/核心技与解放归档.md》《设定/术式吟唱归档.md》

必须检查：
1. 是否把主流超凡体系写成"魔力体系""魔力源体系""魔力模型"
2. 是否把"以太能量"写成神秘者能力来源，而不是常见超凡能量载体
3. 是否把"神秘核心"和"术式"混成同一个概念
4. 是否把月相写成普通神秘者的统一供能源头
5. 是否把月亮体系写成神秘体系的上级或同义词
6. 是否把个人核心技写成可教学术式，或给核心技补吟唱、回路与编号
7. 是否把通用术式漏写分类/编号，或让七十二式编号重复、功能漂移
8. 是否把真名解放写成单纯威力升级，把界临写成无锚点的万能领域，把立序写成无条件言出法随
9. 是否把污染进阶误写成"污染解放"，或混淆污染显相、错误真名与伪解放
10. 是否把主角体系外结果写成真实无吟唱术式，遗漏零核心激活、零以太调用、零术式构筑

风险词表：
```text
魔力|魔力源|魔力模型|魔力衰减|月相.*魔力|月蚀.*魔力|月相.*供能|以太.*核心|专有术式|核心技.*吟唱|星丝术|裂锋术式|污染解放|静默触发式术式|术式下一形态
```

处理原则：
1. 主流超凡体系统一按"神秘体系"处理，个人轴是神秘核心、核心技、真名解放、界临与立序；公共轴是七十二标准式谱及其构筑、吟唱与阵列
2. "以太能量"只能作为常见超凡能量载体、资源、净化或辅助材料使用，不替代神秘核心
3. 月相可以影响反面浮现、边界、旧月残片和神秘输出稳定性，不写成所有普通神秘者的统一能量来源
4. 课堂理论段优先使用"神秘核心、核心显相、核心技、以太载体、通用术式、标准回路、适配与构筑稳定性"等词

输出格式：

```text
【设定术语结论】

【必须改】
1. 位置：
   - 命中词：
   - 冲突口径：
   - 定稿依据：
   - 原位改法：

【建议微调】
1. 位置：
   - 风险：
   - 原位改法：

【可保留】

【下一步】
```
