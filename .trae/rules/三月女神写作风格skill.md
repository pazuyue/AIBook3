# 三月女神写作风格技能（Trae 规则版）

## 文件定位

本 skill 是文风执行入口，不是完整文风规则仓库。

完整文风、样章、硬词表、工具附录与起点高质量网文参考口径，统一以 `设定/文风设定.md` 为唯一来源。本文只保留命令流程、执行快卡、输出格式和修文边界。

## 职责边界

负责：
1. 正文文风、镜头、节奏、对话、打斗质感、去 AI 腔。
2. `/self-revise`、`/fix-prose`、`/daily-style-check` 的执行流程。
3. 将正文校准到“通顺好读、镜头清楚、白话自然、清冷有质感”的项目口径。

不负责：
1. 不改剧情目标、事件结果、战斗规则、设定口径和人物能力。
2. 不替代 `/chapter-plan`、`/review`、`/scene-check`、`/setting-term-check`。
3. 不做 L1/L2/L3 记忆检索、Qdrant 入库、定稿同步。
4. 不保存项目硬设定备份；硬口径、术语、能力、吟唱、当前卷边界按对应权威源执行。

## 文风总目标

正文参考起点高质量网文的成熟读感：读者容易进入、剧情推进明确、爽点兑现清楚、人物声轨稳定、场面镜头可见。

执行时只吸收通俗顺滑、强情节密度、节奏钩子、爽点落地和人物辨识度，不模仿具体作者，不照搬具体作品句式、桥段或专属表达。

最终读感守住一句话：

**清冷但不空，热血但不吵，日常但不水，奇幻但不飘，设定显影不讲穿。**

## 核心执行快卡

### L1 写法底线

1. 先把人、事、场写明白，再压清冷、留白和神秘感。
2. 正文必须像小说段落，不像设定摘要、提示词、审稿意见或说明书。
3. 角色行为、对话和反应要各有辨识度，不能同声道。
4. 战斗必须看得清动作线、战场变化和结果，不能只写抽象气氛。
5. 爽点必须有清楚结果和旁观反应，不能在兑现处继续留白。

### L2 场景调用

1. 日常：关系可见、身体适应可见、异常轻触，不能闲聊灌水。
2. 设定显影：先具体痕迹，再角色判断，再危险后果，不能百科解释。
3. 战斗：动作线清楚，视觉爆点够强，战场变化能看见。
4. 爽点：一个动作一句话，三句内尽量定结果。
5. 过渡：用感官、空间或意象承转，少用“然后”“就在这时”。
6. 对话与视角：按 `设定/文风设定.md` 十一节 9-12 执行——POV 与视角纪律、主角团对话功能位、对话人味检查、高频套路句复查（2026-06-11 合并回收，写任何三轮以上对话前必过）。

### L3 写后自检

1. 删除空话和说明书腔。
2. 压掉重复句式和同构段落。
3. 补至少两个感官通道。
4. 检查主角出手后的遮掩理由。
5. 强化章尾钩子，让下一章变得必要。

## `/self-revise` 与 `/fix-prose`

收到 `/self-revise` 或 `/fix-prose` 时，只做写后自修和原位修文。

允许改：
1. 文风、镜头、动作线、感官、对话、节奏、章尾钩子。
2. AI 腔、说明书腔、句式重复、人物声轨混同、打斗不清。

禁止改：
1. 剧情目标、事件顺序、人物能力、战斗规则、胜负条件。
2. 关键设定、伏笔结论、世界观口径。
3. 已通过设定核查的核心技名、解放语、界临名、术式编号、吟唱句与能力效果词。

发现必须改设定或剧情骨架才能解决的问题时，停止原位硬改，提示用户改用 `/chapter-review`、`/plot-review` 或 `/rewrite-plan`。

### 命令分工

`/self-revise`：整章自修。用于 `/write` 后、`/word-check` 通过后的整章工艺修订，可按问题类型批量修，但不得改变剧情骨架。

`/fix-prose`：局部原位修文。用于用户指定段落、行号、审稿建议或小范围读感问题，每次最多改 5 处。

### 流程闸门

执行 `/self-revise` 前确认当前章已通过 `/word-check`：

1. 低于 4000 字：不得自修，只提示回到 `/write` 补完当前章。
2. 4000-4500 字：只能做轻量修文，优先提示补厚。
3. 4500-6000 字：可进入整章自修。

`/fix-prose` 可用于局部段落，不要求整章已过 `/word-check`，但不得把未完成章节包装成可定稿正文。

### 目标解析

1. 用户给出文件路径时，必须读取并修该文件。
2. 用户贴出正文时，修贴出的正文。
3. 用户没有给路径也没有贴正文时，读取当前卷写作状态中的当前章节，再定位对应正文文件；无法确认目标章节时停止询问。
4. 禁止在不读目标全文或目标段落的情况下盲改。
5. `/fix-prose` 必须有明确段落、行号、审稿项或问题描述；若用户只给整章文件且没有指定问题，只先输出最多 5 条可修位置建议，不直接改文。

### 不可动区域

以下内容即使读感偏硬也不改：

1. 【】内已归档的核心技名、核心引导语、解放语、界临名、立序名、术式名和吟唱句。
2. 已通过设定核查的个人能力或通用术式核心效果与视觉识别词。
3. 断线瞳视觉描写等能力识别标志。
4. 角色辨识度句式。

### 修文优先级

第一层：硬伤，必须改。
1. 字符错误、乱码、漏字。
2. 有害说明书腔、纯规则播报、无上下文元语言解释。
3. 同一段内重复句式。

第二层：节奏，建议改。
1. 过长单句。
2. 连续同构短句。
3. 动作线被解释打断。

第三层：质感，锦上添花。
1. 补缺失的感官通道。
2. 换掉套路反应动作。
3. 压掉冗余定语。

### 输出规则

1. 修文件时，直接写回文件；完成后只输出一句确认：`修文完成，改了 N 处。`
2. 修用户粘贴的正文时，只输出修订后的小说正文，不输出解释、审稿意见、修改说明、标题或清单。
3. 发现设定或剧情层阻塞时，不原位硬改，只简短说明阻塞原因和应改用的流程。

## `/daily-style-check`

收到 `/daily-style-check` 时，只做日常与人味专项检查，不直接修改正文，不替代 `/self-revise`、`/hardword-check` 或 `/setting-term-check`。

检查项：
1. 日常场是否推进关系、情绪、身体适应或暗线。
2. 变身适应是否落到衣物、头发、称呼、被照顾、被注视感、生活规则；不得写成走路、步幅、重心、动作校准或身体调试。
3. 哥伦比娅、林见星、白砚河、苏弥是否各有表达差异和场景功能（功能位标准见 `设定/文风设定.md` 十一节 10）。
4. 中文生活语用是否自然（具体场景例见 `设定/文风设定.md` 十一节 11 第 6 条）。
5. 对话是否落到餐盘、门禁、书页、台灯、衣物、护具、走廊声等场景物上。
6. 普通信息是否被误写成身体感应、指尖发凉、权柄反馈或隔空异常感知。
7. 是否出现诊断式旁白直接分析人物动作，例如"过分端正""用力太均匀""节奏太稳""没有半点迟疑""找不出半点摸索感""不该有的分辨率"；命中时必须给出"动作结果 + 旁人反应 + 角色补救"的原位改法。
8. 三轮以上对话是否通过 `设定/文风设定.md` 十一节 11"对话人味检查"五条+套路句复查（十一节 12）。

输出格式：

```text
【日常风格检查结论】

【必须改】
1. 位置：
   - 问题：
   - 原位改法：

【建议微调】
1. 位置：
   - 风险：
   - 原位改法：

【可保留】

【下一步】
```

## 常见风险提醒

1. 不把“起点高质量”理解成口水化、套路化或纯装逼。
2. 不把“清冷”写成木、不把“热血”写成吵、不把“日常”写成水。
3. 不把设定讲解伪装成对话。
4. 不用规则压死人物真实和场景顺滑。
5. 不为了漂亮句子牺牲读者能否看懂。
