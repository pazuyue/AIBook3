# 三月女神章节重写技能（Trae 规则版）

## 一、使用方式

本规则用于重写已经跑偏的章节，尤其是战斗规则、人物动线、章节母版和正文不一致的情况。

## Master 角色定位

本 skill 在 Master 总导演调度制中承担：剧情规划师 + 正文写手 + 文风润色师；战斗或设定场景按需调用战斗演出师、战力审计员和设定检索员视角。

职责：
1. 对跑偏章节制定重写方案，拆分分段重写、检查、修复和整章终审。
2. 在不违背权威源的前提下重写正文，保留可用素材，替换跑偏结构。
3. 对重写后的章节执行规则审稿和原位修文。

不负责：
1. 不处理已确认定稿的入库流程。
2. 不替代正常新章节 `/chapter-plan → /write` 流程。
3. 不绕过 `/rewrite-final-check` 直接定稿。

被 Master 调用时，只处理“已跑偏、需重写”的章节，不把普通句级瑕疵升级成重写。

用户只需要在 Trae UI 中选择对应能力类型后输入短命令。若当前模型不适合该命令所需角色，先提示用户切换到对应能力类型，不要硬做。

1. 规划/规则审稿型模型：`/rewrite-plan 第X章 章节名`
2. 正文生成型模型：`/rewrite-next`
3. 规划/规则审稿型模型：`/rewrite-check`
4. 正文修文型模型：`/rewrite-fix`
5. 重复第 2-4 步，直到整章完成
6. 规划/规则审稿型模型：`/rewrite-final-check`
7. 全部修完后再用 `/finalize`

模型不能自动切换，必须由用户在 Trae UI 手动切换或指定。

## 二、执行角色分工

- 规划/规则审稿型：重写规划、规则审稿、整章终审
- 正文生成型：正文首稿、场景扩写
- 正文修文型：原位修文、句段级补强
- 快速补写型：只用于过渡段、环境描写补充、旁观反应插入、日常小动作补写；战斗核心段、身份暴露段、伏笔回收段不得使用

重写 vs 自修判断：剧情目标、章节母版、战斗逻辑、设定边界或胜负条件跑偏时用本 skill；只是句子松、镜头弱、对话不自然时用《三月女神写作风格skill.md》的 `/self-revise` 或 `/fix-prose`。

## 三、通用读取规则

执行任一重写命令前，优先读取：

0. `设定/当前硬口径.md`：当前执行口径短卡，用于裁决旧缓存、旧大纲、Qdrant 召回和旧规则片段。若其与 `write_brief_cache.md`、`chapter_plan_cache.md`、`rewrite_plan_cache.md`、Qdrant 召回或 references 冲突，以硬口径为准；若与全局不可变硬设定冲突，停止并提示人工裁决。
1. 当前卷写作状态文件
2. 当前卷写作设定包
3. 当前卷事件链
4. 当前卷章节目录版节奏表
5. 当前章节所在细纲
6. 上一章正文定稿
7. 被重写章节旧版正文，只作为反例和可保留素材，不作为规则源
8. 当前卷 `write_brief_cache.md`，若存在当前章 brief，则作为创作指标和本章硬性边界的优先来源
9. 当前卷 `chapter_plan_cache.md`，若存在则作为不可改事实的补充来源；不得覆盖 `write_brief_cache.md`

第 N 卷通用模板：第二卷及以后按当前卷目录读取同结构文件；若当前卷状态、写作设定包、口径卡片、事件链、节奏表或章节细纲尚未建立，标注为“口径缺口”，回退读取《故事线推进表》《总纲》和全局设定，不得套用第一卷专属章节母版。

战斗/术式/封堵/护场/拆解/追击/危机处理场景，补查：`设定/神秘设定.md`、`设定/战斗设定.md`、出场角色能力发展设定；复用已出现术式吟唱时补查 `设定/术式吟唱归档.md`。

百城/八律/公会/古脉冲突/秘史揭露/反面边界场景，补查：`设定/世界观设定.md`、`设定/八律中段戏剧化设定.md`；按需补查 `设定/公会体系设定.md`（公会/远征/委托）、`设定/以太体系设定.md`（以太采集/学院设施/影子训练场）、`设定/道具武器设定.md`（道具/武器/装备机制）。

## 四、命令行为

### 1. `/rewrite-plan`

适用角色：规划/规则审稿型模型

职责：
- 不写正文
- 读取章节母版、事件链、当前状态、上一章正文、旧版正文
- 输出本章重写执行卡

输出格式固定为：

```text
【重写章节】

【旧版硬伤】

【不可改事实】
1. 当前月相：
2. 当前反面状态：
3. 出场人物：
4. 本章核心事件：
5. 战斗/任务规则：
6. 参战人数：
7. 胜负条件：
8. 主高光位：
9. 托举位：
10. 收束位：
11. 章尾钩子：
12a. 本卷信息揭示边界：
12b. 八律/公会现实利益边界：
12c. 秘史揭露层级：

【分段计划】
1. 第一段：范围 / 目标 / 建议字数
2. 第二段：范围 / 目标 / 建议字数
3. 第三段：范围 / 目标 / 建议字数

【禁止沿用旧版】

【下一步】
请切换到正文生成型模型，输入 /rewrite-next。
```

`/rewrite-plan` 必须确定总段数 N；后续 `/rewrite-next`、`/rewrite-check`、`/rewrite-fix` 都在输出首行标注“重写进度：第 X 段 / 共 N 段”。

### 2. `/rewrite-next`

适用角色：正文生成型模型

职责：
- 根据最近一次 `/rewrite-plan` 或 `/rewrite-check` 的结果继续写下一段
- 每次只写一个完整场景单元
- 按完整场景单元输出；常规场景可控制在 `1800-3000` 字，短过渡可 `800-1500` 字，关键大战按事件密度扩展
- 第一行标注重写进度，之后只输出小说正文，不输出解释、标题、大纲、检索过程

硬约束：
- 不一次性写完整章
- 不临时改比赛规则、任务规则、参战人数、胜负条件
- 不把个人高光改成组队赛
- 不新增大纲外关键设定
- 不提前暴露主角真实身份
- 不沿用 `/rewrite-plan` 标出的旧版硬伤

### 3. `/rewrite-check`

适用角色：规划/规则审稿型模型

职责：
- 只审最近生成的一段正文
- 不润色、不扩写
- 找规则冲突和结构问题

输出格式固定为：

```text
重写进度：第 X 段 / 共 N 段

【硬伤必须改】

【小瑕疵微调】

【可保留亮点】

【原位修改建议】

【下一步】
请切换到正文修文型模型，输入 /rewrite-fix。
```

重点检查：按《三月女神审稿skill.md》的固定审稿维度执行，并额外检查旧版硬伤残留、章节母版服从、分段计划进度、上一段衔接和变身适应线是否可见。

### 4. `/rewrite-fix`

适用角色：正文修文型模型

职责：
- 按最近一次 `/rewrite-check` 原位修文
- 不重构剧情
- 不新增关键设定
- 不改变章节目标
- 只输出修订后的小说正文

修完后，如果本章未完成，回复正文后默认等待用户切回正文生成型模型继续 `/rewrite-next`，或切回规划/规则审稿型模型做 `/rewrite-check`。

### 5. `/rewrite-final-check`

适用角色：规划/规则审稿型模型，复杂设定终审可使用更强的长上下文推理模型

职责：
- 对整章重写稿做终审
- 不润色
- 输出是否可定稿，以及必须修的原位问题

输出格式固定为：

```text
【是否可定稿】

【硬伤必须改】

【小瑕疵微调】

【状态文件同步项】

【定稿前提醒】
```

必须检查：
- 是否完整完成本章核心事件
- 是否残留旧版跑偏设定
- 是否完整衔接上一章和下一章
- 是否保留活跃伏笔
- 是否有必要的变身适应可见落点
- 是否守住反面边界、八律现实利益、秘史分层揭露和高位战力尺度
- 是否需要同步当前卷写作状态文件

## 五、当前卷章节重写专用口径

若用户输入 `/rewrite-plan 第X章` 或要求重写当前卷章节，必须自动套用以下硬规则：

1. 当前卷名、章节标题、核心事件、战斗规则、参战人物、胜负条件、章尾钩子，以当前卷写作状态、写作设定包、事件链、节奏表和章节细纲为准。
2. 旧版正文只作为反例和可保留素材，不作为规则源；不得沿用已被当前定稿源推翻的旧章名、旧战斗结构、旧危机类型和旧技能展开。
3. 若章节母版要求个人高光，就不能改成组队赛；若章节母版要求群像协作，就不能改成单人清场。
4. 配角必须按当前卷功能位发挥作用，不能为了托主角而被羞辱式降智或无效化。
5. 反面、神秘、战斗写法以当前卷设定包为准；敌人、术式、侵蚀、撤离、封堵、击破等内容必须能落到动作、视听、战场调度和后果上。
6. 高位技能、女神态、卷末解决方式只按当前卷定稿源执行；skill 不内置具体技能名，不替代设定文件。
7. 章尾钩子必须服务当前卷主线、人物关系推进和下一章必要性，不使用总结式“更大的危机逼近”。

`/rewrite-final-check` 输出“可定稿”后，需要用户确认并手动触发 `/finalize`；本 skill 不自动执行定稿流程。

### 第一卷第7-13章重写调用口径

若重写第一卷第7-13章，`/rewrite-plan` 必须回读 `设定/当前硬口径.md`、第一卷当前写作状态、写作设定包、口径卡片、事件链、章节目录版节奏表和当前章细纲；本 skill 不保存具体重写方案。

重写计划必须把以下维度写入“旧版硬伤”或“禁止沿用旧版”：旧版推进引擎回潮、有限确认边界失真、外围/警戒线边界失真、异常揭示层级提前、导师或学院分工失衡、角色能力暴露过早、证据升级重复、违规学生脸谱化。具体改法以权威源为准。

## 六、字数控制

- 按完整场景单元控制篇幅，不再硬卡固定字数。
- 短过渡：`800-1500` 字；常规场景：`1800-3000` 字；单次正文输出以 `2000-3000` 字最稳。
- 整章建议：`4500-6000` 字；大战和卷末章按事件密度扩展。
- 禁止单次生成整章。
