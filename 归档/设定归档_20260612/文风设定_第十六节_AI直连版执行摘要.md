## 十六、【工具附录】AI直连版执行摘要

如果要把这份文件浓缩成一段可直接喂给模型的正文提示词，优先抓下面这版：

神秘相关名词、等级、机制、边界、来源与例外，以 `设定\神秘设定.md` 为唯一准绳；本文只约束叙事表达。

你现在要写的是《三月女神今天也只是普通学生》的**可直接入正文的小说文本**。文风要求：通顺好读、镜头清楚、白话自然、清冷有质感；日常有人味，战斗要华丽炫酷、视觉冲击拉满，爽点要一拍定死。主角哥伦比娅必须保持**低调无敌、隐藏身份、内热外冷、关键时刻短暂高位展开**的路线，不能写成高调龙傲天。设定异常信息只能一层层露，不能讲成设定百科；日常必须推进关系、变身适应或异常钩子，不能灌水；打斗必须同时满足**动作线清楚、视觉爆点够强、战场变化看得见、收束镜头够帅**。写作时先把人物、场景、关系、冲突写明白，再往里压氛围和留白；多用动作、站位、视线、感官、对话来带信息，少用总结式解释和抽象判断。遇到拿不准的地方，默认选择更白话、更具体、更像小说正文的写法。

### AI输出硬要求

1. 直接输出正文，不输出说明、分析、提示词解释
2. 不写“接下来”“下面是正文”“这一段表现了”之类提示语
3. 不用括号解释术语，不用像百科一样顺手补设定
4. 不机械追求华丽辞藻，华丽要落在战场视觉、异象、动作结果上
5. 每段都优先让读者看见人、看见场、看见事

### AI输出前最后五问

1. 这段像小说正文吗？
2. 人物说话像本人吗？
3. 画面能看见吗？
4. 节奏会不会拖？
5. 有没有哪一句像设定文、审稿意见或 AI 空话？

### AI输入变量模板

如果要把这份文件拿去直接驱动模型，建议在任务里尽量补齐这些变量：

1. **任务类型**：续写 / 重写 / 原位修文 / 大纲转正文
2. **剧情阶段**：学院前期 / 学院中后期 / 远征期 / 高位展开期
3. **当前场景类型**：日常 / 战斗 / 爽点 / 过渡 / 设定显影
4. **当前出场人物**：谁在场，谁是主视角焦点
5. **本段核心任务**：推关系 / 推异常 / 推战斗 / 推身份反差 / 推秘史 / 推变身适应
6. **本段必须保留的信息**：不能删的事件、台词、设定点、伏笔点
7. **限制条件**：长度、是否要章尾钩子、是否允许改剧情顺序、是否必须接前文气口

变量越完整，输出越稳；变量不完整时，模型按本文件默认规则补全，但优先保守，不乱扩剧情。

### AI长度与密度控制

未明确指定篇幅时，默认按“**完成一个完整场景单元**”来写，不硬凑字数，不草率截断。

默认控制原则：

1. 一个段落只做一件主事：铺场 / 动作 / 判断 / 关系推进 / 爆点收束
2. 一段里不要同时塞满设定、情绪、动作、讲解四层信息
3. 日常段宁可短一点，也不要靠闲聊拖长
4. 战斗段宁可爆点密一点，也不要回合制重复
5. 章尾若要停，优先停在“异常撞上主角 / 判断刚落地 / 危险将临 / 身份险露”的位置

### AI常见失真提醒

模型最容易出现的偏差，调用时默认重点规避：

1. **写成设定说明文**：开头就解释世界观、能力或规则
2. **写成审稿口吻**：出现“这一段表现了”“这里体现出”
3. **写成轻浮日漫腔**：称呼过黏、反应过大、台词太宣言化
4. **写成空洞高级文**：句子好像很美，但没有动作、没有信息、没有推进
5. **写成平均用力**：日常、设定显影、战斗、爽点都一个力度，没有起伏
6. **写成只有结果没有过程**：特别是战斗，只说赢了、压住了、碎了，却没有镜头
7. **写成只顾华丽不顾可读**：形容词很多，但读者看不懂谁在做什么
